# Future HCI v0.5.0-alpha Release Notes

## 🚀 核心特性更新 (New Features)
本版本标志着系统从“知识管理工具”向“学术自动化 Agent”的重大演进。
- **高校 WebVPN 凭证集成**：在 Papers 视图中新增了 RUC WebVPN 的统一登录入口。通过 `persist:ruc-webvpn` 会话分区技术，系统能够在应用内部安全、永久地截获并隔离保存高校内网的身份认证 Cookie。
- **全局下载静默拦截与入库**：针对 WebVPN 独立会话配置了原生的 `will-download` 事件监听。当用户在 Web of Science 或其他数据库中点击下载 PDF 时，系统会：
  1. 强行取消浏览器的系统下载弹窗；
  2. 在后台静默接管文件流并保存至本地 `knowledge_base` 目录；
  3. 自动提取所下载 PDF 的元数据（Title / Authors）；
  4. 通过 IPC 广播事件，在前端 Papers 列表中实现无缝的“秒入库”体验。
- **深度防逃逸窗口管理**：重构了 Electron 的 `setWindowOpenHandler` 逻辑。引入了递归的防逃逸辅助函数 `createSecureVpnWindow`，并解除了严格的跨域限制（`webSecurity: false`）。确保在 Web of Science 跳转至各大出版商（如 Elsevier, Springer）深层页面的过程中，能够100%拦截 `shell.openExternal`，防止链接逃逸至外部系统浏览器，确保 VPN 会话在跨域重定向中的绝对连续性。
- **依赖修复**：修复了因升级至 `pdf-parse v2.4.x` 导致的 API 不兼容（`pdfParse is not a function`）及 JSON 解析报错问题。

## 🔮 下一步演进构想：全自动学术采买 Agent (The Ultimate Vision)

目前，我们已经打通了底层的基础设施：**内网身份认证保留 + 系统级静默下载拦截**（即半自动阶段）。
接下来的核心目标是结合大语言模型（LLM）和无头浏览器（Headless Browser），实现**真正基于自然语言交互的零点击自动化文献采买**。

### 设想工作流
1. **自然语言指令**：用户在全局 AI 对话框中输入：“帮我下载 *Seven HCI Grand Challenges Revisited: Five-Year Progress* 这篇论文”。
2. **意图与参数提取**：LLM 识别指令，提取出关键参数（论文标题/DOI/作者等）。
3. **隐形执行器驱动**：系统在后台拉起一个不可见的隐藏 `BrowserWindow`，该窗口天生继承已有的 `persist:ruc-webvpn`（人大图书馆）高权限会话。
4. **自动化检索与寻址**：
   - Agent 自动导航至高校图书馆的统一发现系统或指定数据库。
   - 注入特定脚本（或集成 Playwright/Puppeteer），自动填入论文标题并触发检索。
   - 在检索结果列表中，利用 DOM 结构分析，智能寻找对应的“PDF 全文”、“Full Text”或“Download”按钮。
5. **触发下载与闭环**：触发点击后，本版本已完成的 `will-download` 静默拦截机制自动生效。几秒钟后，界面直接弹出“下载成功”的通知，论文直接出现在用户的阅读列表中。

### 解决的核心痛点
- **合法合规地突破付费墙**：不依赖不稳定的外部开源免费镜像，而是完全合法地利用用户本校机构购买的数据库资源。
- **免除繁琐的人工跳转**：消除用户从“找论文 -> 挂VPN -> 登录 -> 搜数据库 -> 跨域跳出版商 -> 找下载按钮 -> 选目录保存 -> 拖入软件”的长达数分钟的繁琐劳动，将学术生产力提升至极简的对话体验。
