# OASIS 思想区升级计划 (OneNote 范式对标)

## 1. 摘要
本项目计划将当前的“思想 (Thoughts)”视图升级为类似 OneNote 的多级结构，提升知识管理的维度。核心更新包括：引入独立的分区（Section）侧边栏、为思想条目增加独立的标题字段，并实现基于右键菜单（Context Menu）的交互操作（重命名、删除、移动分区等）。

## 2. 当前状态分析
- **数据结构**：`Thought` 接口目前仅包含 `content`，没有 `title` 或 `category` 字段。标题是前端通过截取正文第一行动态计算的。分区状态不存在。
- **UI 布局**：`ThoughtsView.tsx` 目前为两列布局（左侧 240px 的列表栏，右侧为编辑器）。
- **交互限制**：目前所有的操作（如删除）只能在选中条目后，在右上角的按钮区点击，不支持主流桌面软件的右键快捷菜单。也没有引入任何第三方右键菜单库。

## 3. 具体修改方案

### 3.1 数据层升级 (`src/types/data.ts` & `src/services/dataService.ts`)
- **`Thought` 接口扩展**：
  - 增加 `title?: string` 字段（向后兼容）。
  - 增加 `category?: string` 字段，用于标记归属的分区。
- **`ThoughtsStore` 接口扩展**：
  - 增加 `categories?: string[]`，存储所有自定义分区列表（默认包含一个 `'默认分区'`）。
- **数据服务扩展**：
  - `dataService.ts` 中增加针对分区（增、删、改）以及重命名标题的辅助函数。
  - 兼容旧数据：在 `load()` 时自动将没有分类的思想归入默认分区。

### 3.2 基础组件开发 (`src/components/ui/ContextMenu.tsx`)
- 由于项目坚持原生与极简，将自行实现一个轻量级的 `<ContextMenu />` 组件。
- 该组件通过监听鼠标右键坐标 (`e.pageX`, `e.pageY`) 绝对定位渲染，包含悬浮发光样式的菜单项，并在点击外部时自动销毁。

### 3.3 视图层重构 (`src/components/views/ThoughtsView.tsx`)
- **三列式布局重构**：
  - **列 1 (最左侧)**：新增 `PartitionsSidebar`（约 160px 宽），展示分区列表。支持底部“+ 新建分区”按钮。
  - **列 2 (中间)**：现有的思想列表（240px 宽），但只展示当前选中分区下的思想。
  - **列 3 (右侧)**：现有的编辑器区。如果编辑器的思想设置了 `title`，顶部会展示一个独立的标题输入框（无边框，极简设计）。
- **右键菜单交互集成**：
  - **分区列表右键**：支持【重命名分区】、【删除分区】（提示：将分区内思想移至默认分区）。
  - **思想列表右键**：支持【重命名】、【移动到分区...】、【删除】。

## 4. 关键决策与假设
- **分区删除策略**：为了防止误删大量笔记，删除一个分区时，不会级联删除里面的思想，而是将这些思想的安全归口到“默认分区”。
- **无依赖实现**：不引入额外的 npm 包来实现右键菜单，利用 React State 和 DOM 事件完成，契合项目的极致掌控感。
- **视觉风格**：新增的侧边栏和菜单将继续使用现有的深色玻璃态（Glassmorphism）和发光边框设计。

## 5. 验收标准
1. 数据兼容：旧的思想数据可以正常加载，并自动显示在“默认分区”下，且正文第一行作为备用标题显示正常。
2. 分区管理：能通过右键菜单成功新建、重命名、删除分区。
3. 思想管理：能通过右键菜单成功重命名、移动和删除单个思想。
4. 交互体验：右键菜单在视口边缘不会被遮挡，点击空白处自动消失。
