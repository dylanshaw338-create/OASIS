# 思想区 OneNote 范式对标与验证计划

## 1. 摘要
根据您的要求，进一步完善思想区的功能以对标 OneNote。由于在前一次会话中相关代码（包括分区、独立标题、右键菜单等）已经初步实现，本计划将梳理已完成的架构，并制定后续的验证、测试与细节打磨方案，确保这些功能能够完美、无 bug 地运行。

## 2. 当前状态分析
通过探索代码库，确认以下功能已经实现：
- **数据结构**：`src/types/data.ts` 和 `src/services/dataService.ts` 已扩展 `title` 和 `category` 字段，支持自定义分区列表。
- **UI 布局**：`src/components/views/ThoughtsView.tsx` 已重构为三列布局（分区侧边栏、思想列表、编辑区），并且在编辑区顶部新增了无边框的独立标题输入框。
- **右键菜单组件**：`src/components/ui/ContextMenu.tsx` 已开发完成，实现了基于绝对定位的极简玻璃态悬浮菜单。
- **交互逻辑**：`ThoughtsView.tsx` 中已集成右键新建/重命名/删除分区，以及重命名/移动/删除单条思想的交互。

## 3. 下一步计划：验证与细节完善

虽然基础代码已写好，但为了确保对标 OneNote 的体验完美无缺，计划执行以下验证与修复操作：

### 3.1 核心功能验证
- **右键菜单的边界情况**：验证菜单在屏幕边缘弹出时是否会被遮挡，点击空白区域是否能正确卸载。
- **分区状态管理**：验证在删除当前选中的分区后，焦点是否能正确回退到“默认分区”，且列表能正确刷新。
- **新建逻辑兼容性**：验证在没有任何笔记时，点击“新建分区”或“新建思想”是否会引起崩溃。

### 3.2 UI 与体验打磨
- 检查三列布局在不同窗口尺寸下的响应式表现，确保滚动条样式一致。
- 确保切换分区时，右侧的编辑区能自动聚焦或正确显示空白状态。
- 验证“默认分区”防删除逻辑是否生效。

## 4. 关键决策
- **以验证代替重写**：尊重已有的实现，接下来的工作重心放在测试当前代码的健壮性上。如果发现右键菜单定位不准或分区切换有残影，将直接在现有代码上进行微调。

## 5. 验收标准
1. 鼠标右键能稳定唤出上下文菜单，不出现偏移或闪烁。
2. 分区重命名、删除及思想条目的移动逻辑可以正常保存至本地数据。
3. 界面布局在添加大量分区或思想时依然保持沉浸式玻璃态美感，滚动流畅。
