# OASIS v0.2 版本更新日志 (Release Notes)

## 🌟 核心新特性 (New Features)

### 1. 思想区 (Thoughts) OneNote 范式重构
- **三列沉浸式布局**：全新设计了“分区侧边栏 - 笔记列表 - 编辑区”的三列布局，知识管理层级更加清晰。
- **独立分区 (Section) 管理**：支持多个笔记分区的自由切换，自动记忆并关联对应的笔记。
- **原生级右键菜单 (Context Menu)**：告别简陋的按钮操作，新增全局定制的玻璃态右键菜单。
  - **分区操作**：支持右键新建、重命名、删除分区（安全防误删机制，自动将笔记移至默认分区）。
  - **笔记操作**：支持对单条笔记进行右键重命名、删除、快速移动到其他分区。
- **独立标题支持**：为每条笔记在正文上方增加独立的、无边框的沉浸式标题输入区。

### 2. 交互体验增强 (UX Enhancements)
- **沉浸式输入弹窗 (PromptDialog)**：彻底摒弃 Electron 不支持的原生 `window.prompt`，自研了防拦截、防焦点的全局输入弹窗。
- **焦点智能管理**：切换分区、新建笔记时，焦点会自动且精准地落入对应的输入框；避免打字时发生任何光标劫持。
- **列表优雅换行**：侧边栏笔记预览移除了强制截断，支持最多 3 行的弹性换行显示，确保长标题清晰可见。

### 3. AI 深度融合 (AI Integration) - *（承接近期升级）*
- **全局 AI 悬浮面板**：采用玻璃拟物态 UI，可通过快捷方式随时呼出，支持连续对话。
- **联网搜索与 RAG**：接入 MiniMax，打通本地笔记内容与实时互联网搜索，AI 回答自动通过 Markdown 优雅渲染并剔除生硬的引用标签。

## 🐛 修复的 Bug (Bug Fixes)
1. **输入框无法获取光标**：修复了因 Electron 全局窗口拖拽 (`-webkit-app-region: drag`) 导致的输入框和弹窗无法点击、无光标的问题。
2. **光标诡异跳跃**：修复了因自动保存机制 (`autoSave`) 的 `useEffect` 乱抢焦点，导致用户输入标题时强制跳到正文区的 Bug。
3. **笔记列表排序乱跳**：加入了严格的脏检查 (Dirty Check)，修复了由于单纯点击浏览就更新时间戳，导致笔记在列表中疯狂置顶乱跳的问题。
4. **无法新增/重命名分区**：解决了因 Electron 限制原生弹窗引发的静默崩溃。
