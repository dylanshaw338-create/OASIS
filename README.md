# OASIS — Open Assistance Self Iteration System
## 一种新的人机交互范式探索

> 初稿 · 2026-05-06
> 作者：Dylan Shaw

---

## 命名释义

**OASIS** — *Open Assistance Self Iteration System*

| 词 | 含义 |
|---|---|
| **Open** | 开放的架构与最终开源的愿景；对未来可能性保持开放的姿态 |
| **Assistance** | AI 不是主角，而是辅助者；工具服务于人，而非替代人 |
| **Self** | 极度个人化；系统随使用者本身的认知演化而演化 |
| **Iteration** | 思想、知识、系统本身都处于持续迭代之中；迭代是核心方法论 |
| **System** | 不只是一个工具，而是一套完整的人机协作范式 |

OASIS 在字面上也是"绿洲"——在信息洪流与注意力碎片化的荒漠中，一处可以真正驻足、深度思考的庇护所。

---

## 一、背景与问题

当代主流的人机交互范式建立在几个基本假设之上：

- 界面是**透明的中介**，越薄越好
- 信息应该**随时可得**，触手可及
- 工具应该**服务大众**，可被任何人使用
- 计算在**云端发生**，本地只是终端

这些假设催生了我们熟悉的一切：移动端、SaaS、社交网络、通知推送。它们极大地降低了信息获取的门槛，却也在悄悄侵蚀另一种能力——**深度思考的能力**。

碎片化的信息流、永远在线的协作压力、为最大公约数设计的通用工具，让人类的注意力越来越难以聚焦。我们获取信息的效率在提升，但产生原创思想的能力却在退化。

OASIS 试图探索一个反向的问题：

> **如果我们为"深度思考"而不是"信息消费"来设计人机交互，界面会是什么样的？**

## 二、OASIS 的核心范式

OASIS 的核心可以用一句话概括：

**将计算机变成一个专属的思维空间，而不是一个通用的信息工具。**

这个范式有六个核心特征。

### 2.1 仪式性入口（Ritual Entry）

进入 OASIS 不是一个随意的动作，而是一个有意识的**仪式**。

就像研究者进实验室会换白大褂，禅修者进入冥想室会点燃香烛，进入 OASIS 有一套刻意设计的过渡序列：启动动画、粒子宇宙、打字机文字、短暂的黑暗，最后是主界面的淡入。

这个仪式的功能不是装饰，而是**心理状态的切换**。它在告诉大脑：你即将进入一种不同的模式，专注、深沉、面向未来。

仪式性入口是对"界面越透明越好"这一假设的主动背离。有时候，**阻力本身就是价值**。

### 2.2 本地优先、完全私有（Local-First & Private）

OASIS 的所有数据存储在本地文件系统，不经过任何云端服务器。

这不只是一个技术选择，而是一个哲学立场：**思想属于思考者本人**。

云端服务的商业模式本质上是将用户的数据和注意力变成商品。OASIS 斩断了这条链路。没有算法推荐，没有数据挖掘，没有服务条款约束你的思想归属。

这个空间是你的，完全的，永久的。

### 2.3 思想作为第一类数据（Thought as First-Class Data）

在 OASIS 中，"思想"是核心的数据单元，而不是文档、文件或者笔记。

每个思想有：
- 唯一 ID 与精确的时间戳
- 完整的版本历史（思想的演化轨迹本身就是数据）
- 与其他思想的显式连接关系
- 标签与分类（预留）

这种设计将**思想的演化**变成可观察、可回溯的对象。你不只是在记录结论，而是在记录思考的过程本身。

### 2.4 知识的有机生长（Organic Knowledge Growth）

知识不是线性积累的，它是网状生长的。

双链图谱将孤立的思想节点连接成一个相互引用、相互激发的网络。一个旧的想法可能在六个月后因为一篇新论文而被重新激活，产生新的连接，催生新的洞见。

OASIS 的目标不是存储信息，而是**让知识之间发生化学反应**。

同时，线性的版本迭代视图保留了时间维度：你可以看到自己的思想是如何随着时间演化的，哪些认知发生了根本性的转变，哪些直觉在数年后被证明是正确的。

### 2.5 AI 作为同僚，而非工具（AI as Collaborator）

OASIS 中有三种 AI 角色，它们的定位完全不同：

**知识库 AI**：你自己思想的镜子。它了解你写下的所有内容，能在你需要时检索、关联、总结。它不引入外部信息，只帮你更好地利用自己的知识积累。

**远程工作者**：一个能够执行任务的代理（agent）。通过 SSH 连接到远程算力，它可以帮你运行代码、检索文献、处理数据。它是你意志的延伸，而不是你的替代品。

**思想伙伴**：一个能够进行深度对话的同僚。它的价值不是给出答案，而是通过对话帮你澄清、深化、挑战自己的想法。苏格拉底式的追问，而不是百科全书式的输出。

三种角色的共同点：**它们都服务于你的思考，而不是替代你的思考**。这正是 OASIS 中 *Assistance* 的本质含义。

### 2.6 可扩展的个人范式（Extensible Personal Paradigm）

OASIS 没有固定的终态。它随着使用者的需求和认知而生长——这是 *Self Iteration* 的体现。

每一个新功能模块都是一次 HCI 实验：新的交互方式是否真的降低了思维的摩擦？新的可视化是否真的帮助了认知？OASIS 本身就是研究的对象。

当某些范式成熟后，它们可以被抽象、提炼，以开源的形式释放出去——这是 *Open* 的最终实现，供更广泛的社群探索和迭代。

## 三、与现有范式的比较

| 维度 | 主流范式 | OASIS |
|------|---------|-------|
| 设计目标 | 降低使用门槛 | 加深思考深度 |
| 用户群体 | 最大公约数 | 单一用户（极度个人化） |
| 数据归属 | 平台所有 | 个人所有 |
| 存储位置 | 云端 | 本地 |
| 信息流向 | 推送给用户 | 用户主动构建 |
| AI 角色 | 替代人类任务 | 增强人类思考 |
| 界面哲学 | 透明、无感知 | 有仪式感、有存在感 |
| 知识结构 | 线性/层级 | 网状/有机 |
| 演化方式 | 产品迭代 | 个人认知迭代 |

## 四、未解决的问题

OASIS 目前仍面临一些开放性问题，它们本身也是值得探索的研究方向：

1. **沉浸感与可及性的张力**：仪式性入口会否在某些情境下变成障碍？如何设计可调节的沉浸深度？

2. **个人化与知识传播的矛盾**：极度私人化的知识系统如何在需要时对外分享？什么样的抽象层次适合开源？

3. **AI 协作的边界**：当 AI 开始影响你的想法生成过程，你的思想还在多大程度上是"你的"？

4. **长期可维护性**：一个完全个人化的系统在十年、二十年后是否仍然可用？如何设计数据格式以保证长期可读？

5. **认知负荷的双刃剑**：丰富的知识图谱可能在某些状态下反而造成认知过载。如何在"全局视野"和"专注当下"之间切换？

---

## 五、这不是什么

最后，明确 OASIS **不试图解决的问题**：

- 它不是一个生产力工具，不追求任务完成的效率
- 它不是一个协作平台，不为团队工作设计
- 它不是一个内容发布系统，不为展示而存在
- 它不是对现有笔记软件的替代，而是探索一个不同的问题

OASIS 是一处**思维的绿洲**，一个让深度探索得以发生的庇护空间。
